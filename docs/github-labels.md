# MapleCard GitHub Labels

Use these suggested labels to keep demo feedback triage consistent across issues
and sprint follow-up work. This is documentation only; do not treat it as a
requirement to create labels automatically.

## Type Labels

- `type: parser`
- `type: clarification`
- `type: frontend-ux`
- `type: backend-api`
- `type: staging`
- `type: known-limitation`
- `type: real-data-gap`

## Priority Labels

- `priority: p0-demo-blocker`
- `priority: p1-core-flow`
- `priority: p2-confusing-ux`
- `priority: p3-catalog-wording`
- `priority: p4-future-enhancement`

## Status Labels

- `status: needs-repro`
- `status: accepted`
- `status: known-limitation`
- `status: converted-to-sprint`
- `status: closed-after-review`

## Suggested Usage Notes

- apply one primary `type:` label per issue whenever possible
- pair each issue with one `priority:` label during review
- use `status:` labels to show the current decision after triage
- prefer consistency over label volume so the tester round remains easy to scan
