# MapleCard Tester Invite Script

## Plain-English Intro

MapleCard is a staging demo that turns a plain shopping list into an initial
store recommendation, alternative options, and clarification questions when your
intent is ambiguous.

We are using this tester round to learn whether the shopping flow feels clear,
useful, and trustworthy before adding real inventory, checkout, accounts, or
production-scale systems.

Staging link: https://maple-card.vercel.app

Estimated time: 10 to 15 minutes.

## What Testers Should Try

- a simple yogurt request
- a simple coffee request
- duplicate yogurt lines to confirm the questions stay separate
- a normal grocery list
- empty input or an intentionally incomplete request to see error handling

## What Testers Should Not Expect Yet

- no real prices
- no checkout
- no real retailer inventory
- no saved accounts

## Privacy Note

- copied feedback reports exclude shopping-list text by default
- testers can opt in to include shopping-list text when it is necessary to
  explain an issue

## Feedback Ask

Please send the copied feedback report after your session, or file a GitHub demo
feedback issue if that is easier for you.

## Longer Version For Email

Subject: MapleCard controlled tester round invite

Hi,

I’m inviting a small group of testers to try MapleCard in staging. MapleCard is
a shopping-intelligence demo that turns a plain shopping list into an initial
store recommendation, alternative options, and clarification questions when the
input is ambiguous.

The goal of this round is to learn whether the flow feels understandable,
useful, and trustworthy before we add real inventory, checkout, saved accounts,

- yogurt
- coffee
- duplicate yogurt lines
- a normal grocery list
- empty input or an error-handling case

Staging link: https://maple-card.vercel.app

Please note that MapleCard does not have real prices, checkout, real retailer
inventory, or saved accounts yet. The current demo uses synthetic data for this
round.

When you finish, please copy the feedback report from the app and send it back,
or file a GitHub demo feedback issue if that is more convenient. The copied
feedback report excludes shopping-list text by default, and you can opt in to
include it only if it helps explain an issue.
